# 工作审查检查清单

配合 `criticism-self-criticism` skill 和 `self-critic` agent 使用。在完成阶段性工作后逐项检查。

## 一、完整性检查

- [ ] 原定目标是否全部达成？
- [ ] 是否有遗漏的需求或子任务？
- [ ] 边界情况和异常路径是否处理？
- [ ] 相关文档/注释是否同步更新？
- [ ] 是否有"先跳过后面再补"但实际未补的内容？

## 二、正确性检查

- [ ] 核心逻辑是否经过验证（测试/运行/手动检查）？
- [ ] 修改是否引入了新问题？（回归风险）
- [ ] 假设和前提条件是否仍然成立？
- [ ] 数据流和状态变更是否正确？
- [ ] 并发/竞态条件是否考虑？（如适用）

## 三、方法论检查

- [ ] 是否做了充分的调查研究再行动？（`investigation-first`）
- [ ] 是否抓住了主要矛盾？（`contradiction-analysis`）
- [ ] 方案是否经过实践验证？（`practice-cognition`）
- [ ] 是否收集了相关方的意见？（`mass-line`）
- [ ] 资源是否集中在最重要的问题上？（`concentrate-forces`）
- [ ] 多个目标之间是否做了平衡？（`overall-planning`）

## 四、质量检查

- [ ] 代码/方案是否清晰可读？
- [ ] 是否存在不必要的复杂性？
- [ ] 是否有"差不多就行了"的敷衍之处？
- [ ] 是否有已知但选择忽略的问题？
- [ ] 如果重新来过，会在哪里做出不同选择？

## 使用方式

1. 完成工作后，逐项过一遍
2. 对未通过的项，评估严重程度：
   - **必须改正**（对抗性矛盾）：阻碍目标达成或引入重大风险
   - **应当改正**（非对抗性矛盾）：影响质量但不致命
   - **建议改进**（发展建议）：可以做得更好
3. 对必须改正的项，立即修复
4. 对应当改正的项，制定修复计划
5. 记录本次审查发现的模式，作为下次工作的重点关注
